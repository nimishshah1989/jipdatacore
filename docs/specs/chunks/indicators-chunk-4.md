# Chunk 4 — TA-Lib oracle + golden-file tests

**Complexity**: M
**Blocks**: chunk-5
**Blocked by**: chunk-3

## Goal
Establish a correctness baseline by comparing pandas-ta-classic outputs against TA-Lib (the industry reference) on a frozen dataset. Create golden fixtures, a CI-only Dockerfile that installs TA-Lib for fixture regeneration, and a test that asserts parity.

## Files
- **Create**: `Dockerfile.talib-oracle`
  - Base: `python:3.11-slim`
  - Installs TA-Lib C library: `apt-get install -y build-essential wget && wget http://prdownloads.sourceforge.net/ta-lib/ta-lib-0.4.0-src.tar.gz && tar xzf ... && ./configure && make && make install`
  - Installs: `pip install TA-Lib pandas numpy pyarrow`
  - Entrypoint: runs `scripts/regenerate_talib_golden.py`
- **Create**: `scripts/regenerate_talib_golden.py`
  - Loads `tests/computation/fixtures/golden/nifty50_2023_2025.parquet` (OHLCV)
  - Runs TA-Lib: RSI(14), MACD(12,26,9), ADX(14), BBANDS(20,2), ATR(14), OBV, SMA(50), EMA(20)
  - Writes `tests/computation/fixtures/golden/nifty50_indicators_talib.parquet`
  - Manual step: run once per pandas-ta version bump
- **Create**: `tests/computation/fixtures/golden/nifty50_2023_2025.parquet`
  - OHLCV for NIFTY 50 constituents, 2023-01-01 to 2025-12-31
  - Pulled from `de_equity_ohlcv` via a one-off extraction script (document in test docstring)
  - Columns: `date, instrument_id, symbol, open, high, low, close, close_adj, volume`
  - ~50 symbols × 750 trading days ≈ 37,500 rows, ~2 MB parquet
- **Create**: `tests/computation/fixtures/golden/nifty50_indicators_talib.parquet`
  - Generated by `regenerate_talib_golden.py`
  - Keyed on `(instrument_id, date)`, columns: `rsi_14_talib, macd_line_talib, macd_signal_talib, macd_hist_talib, adx_14_talib, bb_upper_talib, bb_middle_talib, bb_lower_talib, atr_14_talib, obv_talib, sma_50_talib, ema_20_talib`
- **Create**: `tests/computation/test_indicators_v2_golden.py`
  - Loads both fixtures
  - Runs pandas-ta-classic on the OHLCV fixture via the engine
  - Asserts per-row parity against the TA-Lib fixture within tolerance
  - Tolerance: `1e-6` for RSI/SMA/EMA/MACD, `1e-4` for ADX (Wilder smoothing can diverge very slightly), `1` for OBV (integer accumulator can differ in first row handling)
  - Asserts no NaNs in row index 300+ (past warmup)
  - Asserts NaNs present in early rows (RSI_14 should be NaN for rows 0-13)
  - Asserts Decimal quantization: loaded from DB as strings, confirm step is exactly `0.0001`

## Acceptance criteria
- `pytest tests/computation/test_indicators_v2_golden.py -v` all green
- If ADX fails at 1e-4 → relax to 1e-3 and document in test (not a blocker)
- `Dockerfile.talib-oracle` builds successfully (one-off verification, not in CI runtime)
- Golden fixtures committed to git LFS or regular git (if <5 MB each)

## Verification commands
```bash
# One-off fixture regeneration (optional, only if pandas-ta version changes)
docker build -f Dockerfile.talib-oracle -t talib-oracle .
docker run --rm -v $(pwd):/work talib-oracle

# Runtime test (no TA-Lib required)
pytest tests/computation/test_indicators_v2_golden.py -v
```

## Fallback
If TA-Lib install in Docker is too painful and you can't get the fixture built in a reasonable time, use published reference values from `pandas-ta-classic`'s own test suite as the baseline instead — ship a weaker but actionable golden test, and file the TA-Lib oracle as a follow-up.
